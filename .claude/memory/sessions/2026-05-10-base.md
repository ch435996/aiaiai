## 3. 关键 Bug 与修复

### 3.1 ChatOrchestrator NPE

- **现象**: `ChatMemoryService.getOrCreateChatMemory()` NPE
- **原因**: LangChain4j 1.0.0 中 `AiServices.builder().chatMemory()` 只支持单 session，多 session 需用 `chatMemoryProvider()`
- **修复**: 改用 `chatMemoryProvider(memoryId -> MessageWindowChatMemory.builder()...)`，全局单个 Assistant 实例

### 3.2 Embedding API 网络不通

- **现象**: `ConnectException` 连接 `api.openai.com`
- **原因**: 国内网络无法直连 OpenAI API
- **修复**: 改用阿里云百炼平台
  - `base-url`: `https://dashscope.aliyuncs.com/compatible-mode/v1`
  - `model-name`: `text-embedding-v3`
  - API key: ``

### 3.3 Pinecone 维度不匹配

- **现象**: `Vector dimension 1024 does not match the dimension of the index 1536`
- **原因**: 百炼 `text-embedding-v3` 输出 1024 维，但 Pinecone 索引创建时用了 1536 维
- **探索**:
  - 尝试设置 `dimensions=1536` 让百炼输出 1536 维 → 百炼报错：仅支持 `[64,128,256,512,768,1024]`
- **最终方案**: 删除 Pinecone 索引重建，配置改为 1024 维
  - `embedding.dimensions: 1024`
  - `pinecone.dimension: 1024`
  - `PineconeServerlessIndexConfig` 自动创建新索引

## 6. LangChain4j 1.0.0 API 关键发现

| 假设（基于新版文档） | 1.0.0 实际情况 |
|---------------------|---------------|
| `@AiService` 注解 | **不存在** — 接口为纯 POJO，`AiServices.builder()` 动态代理 |
| `ChatLanguageModel` | 已重命名为 `ChatModel` |
| `ChatMemoryStore` 接口 | 存在，位于 `dev.langchain4j.store.memory.chat` |
| `@Tool` 的 `value` 是 `String` | 实际是 `String[]` |
| `.chatMemory()` 多 session | 仅支持单 session，需用 `.chatMemoryProvider()` |
| `PineconeServerlessIndexConfig.metric()` | beta5 版已移除该方法 |
| `OpenAiEmbeddingModel.dimensions()` | 存在，可指定输出维度 |
